# VolleyStats — Трекер задач и бизнес-требований

> **Для агентов**: перед началом любой новой задачи — прочитать этот файл.
> Если новая задача дублирует или конфликтует с существующей — сообщить пользователю до начала работы.

## ✅ Реализовано

### Инфраструктура
- [x] Монорепо (pnpm + Turborepo): `apps/web` + `apps/api` + `packages/shared`
- [x] PWA (vite-plugin-pwa, Workbox, autoUpdate, offline-first IndexedDB)
- [x] Backend API (Hono + SQLite, WAL mode, порт 3002)
- [x] Деплой: фронтенд → nginx `/var/www/volleystats/`, API → PM2 `volleystats-api`
- [x] Фоновая синхронизация IndexedDB ↔ сервер (timestamp-merge стратегия)
- [x] PNG-иконки для PWA (icon-192.png, icon-512.png)
- [x] Pinch-to-zoom разрешён (убран `user-scalable=no`)

### Ввод матча (live.tsx)
- [x] Горизонтальная площадка: домашние (3×2) | СЕТКА | соперник (3×2, зеркально)
- [x] Rally FSM: `IDLE → SERVE → RECEPTION → IN_PLAY → RALLY_OVER`
- [x] Подача: 2 кнопки «В игру» / «Ошибка» для обеих команд (quality ретроактивный)
- [x] Приём домашней команды: 7 кнопок (++ / + / ! / − / / / = / Ошиб)
- [x] Приём соперника: 3 кнопки (+ / 0 / Ошиб)
- [x] Атака in_play: «+» (очко) / «В игру» (нейтрал) / «−» (ошибка с подменю)
- [x] Подменю ошибок атаки: Аут / В сетку
- [x] Блок: «+» (очко) / «−» (ошибка) / «с» (смягчение) / «0» (касание) — зависит от зоны
- [x] Защита: «+» (поднял) / «−» (ошибка)
- [x] Ретроактивное обновление quality подачи (по результату приёма)
- [x] Ретроактивное обновление quality атаки (по блоку/защите соперника): `attack_kill`, `attack_blocked`
- [x] Ротация при side-out + автозамена либеро при попадании на переднюю линию
- [x] Undo: внутри розыгрыша (undoLastRallyAction) и между розыгрышами (undoLastRally)
- [x] Кнопка «Начать розыгрыш» (без плашки «Ошибка соперника +1» — убрана)

### Либеро и замены
- [x] До 2 либеро в команде, только 1 на площадке одновременно
- [x] LiberoControl: вывод статуса, кнопки «На площадку» / «Убрать»
- [x] Блок либеро отображается на половине домашней команды (левая колонка)
- [x] Автозамена при ротации + откат при undo
- [x] SubstitutionModal: двухшаговый (кто выходит → кто входит), макс 6 на сет
- [x] Правило обратной замены
- [x] Скамейка домашней команды: список игроков не в составе (под половиной домашних)

### Статистика и экспорт

## Архитектурные решения (почему так)

| Решение | Причина |
|---------|---------|
| Quality ретроактивно | Реальный процесс ввода — сначала подача, потом видно качество по приёму |
| Away = generic players | Не нужна реальная статистика соперника на данном этапе |
| LiberoControl только home | Away libero не отслеживается |
| `attack_kill` через defense error | Атака без `result='success'` — очко определяется ответным действием соперника |
| Ошибка соперника убрана из UI | Засоряла основной флоу; планируется альтернативный ввод |
